## 1. Motivation and significance

ColaboraPANC addresses the need for software-supported, auditable workflows for PANC mapping and validation, where community observations, scientific review, and environmental context must coexist.

## 2. Software description

The software combines a Django/DRF backend, domain services for enrichment/alerts/prioritization, and an Expo/React Native mobile client.

## 3. Architecture

Core modules include backend application services (`mapping/`), project configuration (`config/`), mobile app (`mobile/`), and test/documentation assets.

## 4. Workflow

Typical workflow: data contribution -> AI-assisted suggestion -> human expert review -> validated record -> downstream environmental/territorial analysis.

## 5. Example use case

An evaluator can set up the backend locally, run tests, and inspect the map/review endpoints to validate software behavior for collaborative scientific curation.

## 6. Impact and reuse potential

Potential reuse includes biodiversity informatics, nutrition/agroecology extension projects, and community-driven environmental intelligence pipelines.

## 7. Limitations

Integration branches that depend on external APIs may require credentials and are subject to external provider availability.

## 8. Availability and reuse

Source code is available in the public repository under MIT license.

## 9. Data statement

No fabricated scientific dataset is included; data packaging and external deposit decisions remain author-controlled.
